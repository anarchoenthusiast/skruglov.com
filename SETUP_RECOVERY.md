# Руководство по восстановлению и развертыванию проекта (Astro Blog)

Этот документ содержит пошаговую инструкцию по развертыванию проекта на новой машине.

## 1. Предварительные требования

Перед началом работы убедитесь, что установлены следующие инструменты:

1.  **Git**: Для клонирования репозитория.
2.  **Runtime**:
    *   **Bun** (рекомендуется, так как в проекте есть `bun.lock`): [Установка Bun](https://bun.sh/)
    *   *Альтернатива*: Node.js (v18+).
3.  **1Password CLI (`op`)**: Для восстановления секретов.
    *   [Инструкция по установке](https://developer.1password.com/docs/cli/get-started/)
    *   После установки войдите в аккаунт: `op signin`

## 2. Установка проекта

1.  **Клонирование репозитория:**
    ```bash
    git clone <your-repo-url> astro-blog
    cd astro-blog
    ```

2.  **Восстановление секретов (.env):**
    Мы используем скрипт для подтягивания конфигов из 1Password.
    ```bash
    ./scripts/secrets-manager.sh restore
    ```
    *Если скрипт сообщает, что секреты не найдены, это нормально для нового проекта. Создайте `.env` файл вручную на основе примера, если он потребуется.*

3.  **Установка зависимостей:**
    ```bash
    bun install
    # Или, если используете npm:
    # npm install
    ```

## 3. Запуск и разработка

### Локальный сервер разработки
Запускает проект в режиме hot-reload по адресу `http://localhost:4321`.

```bash
bun run dev
```

### Сборка (Build)
Создает оптимизированную версию сайта в папке `dist/`.

```bash
bun run build
```

### Предпросмотр сборки (Preview)
Запускает локальный сервер для проверки собранной версии (`dist/`).

```bash
bun run preview
```

## 4. Деплой

Проект настроен для деплоя на Netlify (см. `netlify.toml`).
Также поддерживается Vercel (см. `astro.config.mjs` или наличие адаптеров).

Для деплоя на Netlify используется команда:
```bash
npm run build:netlify
```

## Структура важных файлов

*   `src/content/`: Markdown/MDX файлы с контентом блога и проектов.
*   `src/pages/`: Роутинг Astro.
*   `src/components/`: UI компоненты.
*   `astro.config.mjs`: Конфигурация Astro.
*   `scripts/secrets-manager.sh`: Утилита для бэкапа/восстановления `.env`.

## Бэкап секретов (для разработчика)

Если вы добавили новые ключи в `.env`, сохраните их в 1Password:

```bash
./scripts/secrets-manager.sh save
```
